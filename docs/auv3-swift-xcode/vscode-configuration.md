# Configuring VSCode for AUv3 Project with Swift and C++

Here are some notes on how to enable error highlighting and code completion for AUv3 projects. It's assumed that the codebase is a templated project for `auv3-swift-xcode`, but it would also be applied for general AUv3 projects that mix Swift and C++.

Tested on the following environment:

- Mac Mini M4
- MacOS 26.3
- VSCode 1.113.0

## Extensions

- clangd
- Swift Language Support for Visual Studio Code
- Sweetpad

## .vscode/settings.json

```json
{
  "sweetpad.build.xcodeWorkspacePath": "Project1.xcodeproj/project.xcworkspace"
}
```

This specifies which workspace SweetPad targets when it calls xcodebuild internally.

## buildServer.json

```json
{
  "name": "xcode build server",
  "version": "1.3.0",
  "bspVersion": "2.2.0",
  "languages": ["c", "cpp", "objective-c", "objective-cpp", "swift"],
  "argv": ["/opt/homebrew/bin/xcode-build-server"],
  "workspace": "/Users/user/Documents/projects/sonic/templates/auv3-swift-xcode/Project1.xcodeproj/project.xcworkspace",
  "build_root": "/Users/user/Library/Developer/Xcode/DerivedData/Project1-aaabbbcccdddeeefffggghhhiiij",
  "scheme": "Project1Extension",
  "kind": "xcode"
}
```

Press CMD+Shift+P and run `SweetPad: Generate Build Server Config (buildServer.json)` to generate the file.

clangd and Swift LSP use `xcode-builder-server` to provide completion, jump, and error analysis features within the IDE.
This JSON file provides the necessary configuration to run `xcode-builder-server`.

## build

This includes cached C++ headers and modules automatically generated from Swift, as well as the paths to framework files. It is required for resolving C++ include paths.

It is generated by building with xcodebuild.

```sh
	xcodebuild -project Project1.xcodeproj -scheme Project1Extension -destination 'platform=macOS' -derivedDataPath build build
```

In theory, the builderServer configuration alone should be sufficient, but there are some cases where the include path cannot be resolved, so I have added additional settings to ensure that includes can be resolved.

## Summary

By configuring these settings, error highlighting, jump to definition, and code completion will work for both Swift and C++ code.
